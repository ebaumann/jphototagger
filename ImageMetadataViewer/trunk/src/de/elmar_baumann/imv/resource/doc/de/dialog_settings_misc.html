Protokolloptionen (Logdatei)

Der Programmablauf wird protokolliert und das Protokoll in eine Logdatei geschrieben. Diese kann angezeigt werden durch Klick auf eine Schaltfläche am unteren Fensterrand. Sie hat ein grünes Häkchen, wenn alles ok ist und ein rotes Kreuz, falls Fehler auftraten. Bedeutung der Auswahl:

  • WARNUNG: Nur wichtige Fehler und weniger wichtige werden protokolliert; im Normalfall ist das die günstigste Einstellung
  • SCHWERWIEGEND: Nur wichtige Fehler werden protokolliert
  • INFO: Informative Ausgaben, für den Benutzer eines Programms meist unwichtig
  • KONFIG: Ausgaben betreffend die Konfiguration, für den Benutzer eines Programms meist unwichtig
  • FEIN: Detaillierte Ausgaben, insbesondere SQL-Statements, die die Datenbank aktualisieren
  • FEINER: Noch detailliertere Ausgaben, insbesondere SQL-Statements, die die Datenbank abfragen (und nicht modifizieren)
  • AM FEINSTEN: Alles wird protokolliert

Wählen Sie WARNUNG, solange keine Probleme bestehen und zur Fehlerbehebung AM FEINSTEN. Meldungen, die keine Warnungen sind oder keine wichtigen Fehler, werden in die Standardausgabe geschrieben, andernfalls würde die Protokolldatei viele Megabyte groß (erfahrene Computerbenutzer wissen: Die Standardausgabe lässt sich umleiten in eine Datei). Die Standardausgabe sehen Sie im Fenster, über das das Programm Bilder gestartet wurde. Unter Windows ist es minimiert in der Taskleiste.

Ordner der Kamera-Speicherkarte

Hier können Sie einen Ordner auswählen, von dem alle Bilder einer Speicherkarte in einen Ordner kopiert werden.

Wählen Sie den Ordner aus, in dem der Inhalt Ihrer Kamera-Speicherkarte angezeigt wird, wenn Sie Ihre Kamera mit dem Rechner verbinden oder die Karte in einen Kartenleser einführen: Unter Windows einen Laufwerksbuchstaben, unter Linux einen Ordner, beispielsweise /mnt/compactflash. Das Laufwerk / der Ordner werden nach Bildern durchsucht einschließlich aller Unterverzeichnisse.
