Einstellungen, Kartenreiter "Sonstiges"

Versteckte Ordner einlesen und anzeigen

Wollen Sie Bilder archivieren, die in versteckten Ordnern gespeichert sind, kreuzen Sie dieses Kästchen an.

Nach dem Programmstart automatisch auf eine neue Version prüfen

Ist diese Option angekreuzt, sucht JPhotoTagger im Internet nach einer neueren Programmversion. Existiert eine, wird diese zum Download angeboten.

Schaltfläche Suchen anzeigen

Falls angekreuzt, ist eine Schaltfläche neben dem Eingabefeld für die Schnellsuche. Das ist die Standard-Einstellung. Sie sparen Platz durch Entfernen des Häkchens (wirkt sich erst aus nach Neustart). Mit der Enter-Taste können Sie eine Suche ebenso starten: Sie tippen einen Suchbegriff ein und betätigen anschließend die Enter-Taste.

Beim Export von GPS-Standorten den Dateinamen exportieren

Ist diese Option angekreuzt, wird beim Export von GPS-Standorten zusätzlich der Dateiname (ohne Verzeichnispfad) in den Titel (Namen) exportiert.

Experimentelle Dateiformate erlauben

Ist diese Option angekreuzt, werden Bilder mit Dateiformaten archiviert, die nicht "offiziell" unterstützt werden. Es wird versucht, deren EXIF-Daten auszulesen (Aufnahmedatum, Blende, Verschlusszeit, etc.) und falls eingestellt, deren IPTC-Daten. Dies könnte jedoch nicht gelingen. Betroffen sind auch eingebettete Vorschaubilder. Sie sollten in diesem Fall die Vorschaubilder von einem externen Programm erzeugen lassen, wie hier beschrieben.

Die Bildformate wurden auf den Rechnern der Entwickler getestet. Es könnten unerwartete Fehler auftreten außer (den erwarteten Fehlern) dass keine EXIF-Daten gelesen werden und keine Vorschaubilder erzeugt werden können. Bitte benachrichtigen Sie in diesem Fall den Support über den Menüpunkt Hilfe > Fehler melden.

Sie können auf jeden Fall Stichwörter und andere Metadaten für diese Dateien eingeben und diese so finden, auch wenn JPhotoTagger keine EXIF-Daten lesen und Vorschaubilder erzeugen kann.

Kopieren/Verschieben von Dateien: Quelldatei

Nachfragen, falls gleichnamige Datei im Zielordner existiert

Ist diese Option aktiv, wird erst nachgefragt, ob eine existierende Zieldatei überschrieben werden soll, bevor die Datei kopiert wird. Beim Verschieben hat diese Option keine Auswirkung.

Automatisch umbenennen , falls gleichnamige Datei im Zielordner existiert

Ist diese Option angekreuzt, wird die Quelldatei automatisch umbenannt, falls im Zielordner eine gleichnamige Datei existiert: Vor der Endung wird ein Unterstrich angehängt gefolgt von einer fortlaufenden Nummer.

Zeichensatz der IPTC-Daten

Stellen Sie hier ein, mit welchem Zeichensatz die IPTC-Daten kodiert sind. Leider schreiben dies nicht alle Programme (fehlerfrei) in die IPTC-Daten. Ist der Zeichensatz falsch, werden beispielsweise Umlaute verkehrt angezeigt und Sie können nicht danach suchen, außer, Sie geben die gleichen "Hieroglyphen" ein, die falsch angezeigt werden.

Was / wie viel soll protokolliert werden?

Der Programmablauf wird laufend protokolliert, Sie können das Protokoll anschauen über Fenster > Programmausgaben. Was darin angezeigt wird, stellen Sie hier ein. Bedeutungen:

  • SCHWERWIEGEND: Nur wichtige Fehler werden protokolliert. Schwer wiegendes wird bei allen anderen Einstellungen zusätzlich protokolliert.
  • WARNUNG: Nur wichtige Fehler und weniger wichtige werden protokolliert. Warnungen werden bei anderen Einstellungen zusätzlich protokolliert außer bei SCHWERWIEGEND.
  • INFO: Informative Ausgaben, beispielsweise welche Verzeichnisse gerade automatisch eingelesen werden. Dies ist die Grunddeinstellung.
  • KONFIG: Ausgaben betreffend die Konfiguration, für den Benutzer eines Programms meist unwichtig.
  • FEIN: Detaillierte Ausgaben über den Programmablauf
  • FEINER: Noch detailliertere Ausgaben, insbesondere SQL-Statements, die die Datenbank aktualisieren
  • AM FEINSTEN: SQL-Statements, die die Datenbank abfragen und nicht modifizieren, noch detailliertere Informationen über den Programmablauf

Wählen Sie WARNUNG oder INFO (Voreinstellung), solange keine Probleme bestehen und zur Fehlerbehebung AM FEINSTEN.

Unabhängig davon werden zwei Logdateien geschrieben: Ein Fehlerprotokoll, das nur Fehler enthält (WARNUNG) und eines mit allen Ausgaben (AM FEINSTEN). Diese Protokolle können angezeigt werden über das Menü Fenster > Fehlerprotokoll und Fenster > Protokoll mit allen Ausgaben.

Datenbankordner

Sie können die Datenbank in einem anderen Ordner anlegen lassen. Wollen Sie Ihre bisherigen Daten übernehmen, verschieben Sie aus dem aktuellen Datenbankordner, der im Dialog angezeigt wird, folgende Dateien und Ordner - sofern vorhanden - nach Beenden von JPhotoTagger und vor einem Neustart in den neuen Ordner:

  • database.* (database.backup, database.data, database.log, database.properties, database.script)
  • Ordner thumbnails
  • Ordner ExifCache
  • Ordner EmbeddedXmpCache
  • Ordner IptcIgnoreCache
  • Ordner plugincache

Sie könnten die Datenbank auslagern wollen in einen bestimmten Ordner, den Sie regelmäßig sichern (Backup). Und/oder Sie speichern die Datenbank in einem Netzwerk, in das Sie sich von verschiedenen Rechnern aus anmelden, wobei die Ordnerpfade zu den Bildern auf allen Rechnern gleich lauten.

Diese Option ist nicht dafür gedacht, dass mehrere Benutzer die gleiche Datenbank teilen! Beim Start meldet JPhotoTagger, falls die Datenbank (wahrscheinlich) geöffnet ist. Sind Sie sicher, dies ist nicht der Fall, können Sie JPhotoTagger starten. Im günstigsten Fall verhindert das Betriebssystem, dass die gleiche Datenbank zwei Mal geöffnet wird, im ungünstigsten ist die Datenbank anschließend beschädigt - es ist nicht vorherzusagen, was genau geschieht. Falls Sie sicher sind, die Datenbank wird nicht gleichzeitig mehrmals geöffnet, können Sie diese mit mehreren Benutzern teilen, zu empfehlen ist dies nicht (In diesem Fall sollten die Ordnerpfade zu den Bildern bei allen Benutzern identisch sein).

Ordner für Datenbanksicherungen

In diesem Ordner legt JPhotoTagger Unterordner an, in die Datenbanken gespeichert werden, falls Sie den Menüeintrag Datei > Datenbank sichern auswählen. Standard ist der gleiche Ordner, in dem die Datenbankdateien gespeichert sind.
