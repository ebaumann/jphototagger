Models reflecting database content and tied to views (lists, trees, ...).

Database models

To reflect the current database content, a model is a database listener and synchronizes its content with the database on database updates.

A model itself shall not provide database change methods for re-using database change methods in other contexts. Also the same database content can be mirrored by different models, e.g. a list model and a combo box model too.

Other models

Some models do not reflecting database content, e.g. a model with logfile formatter classes for controlling the output formatting of loggers.
