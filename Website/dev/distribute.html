How to distribute JPhotoTagger

A JPhotoTagger distribution consists of:

  • Platform independend ZIP file
  • Windows setup (EXE) file
  • Up to date version information file jphototagger-version.txt; JPhotoTagger auto update checks it for newer versions
  • Up to date download web page containing the current JPhotoTagger version and HTML meta date
  • Up to date .htaccess file, which redirects from the download web page to the current download location

Everything is done automatically through the Ant file distribute.xml within the repository root, subdirectory DeveloperSupport/script. Each task can be executed separately by calling an appropriate Ant target or completely by an Ant .*-all-.*-target.

Some targets reading private properties from property files in DeveloperSupport/conf which may differ from system to system and excluded from version control. They must exist and containing the key value pairs as described below:

  • googlecode.properties with the keys username and password (login data, not very secure, restrict access!)
  • innosetup.properties with the key compil32.exe (file path to Inno Setup compiler, e.g. C:\\Program Files\\InnoSetup\\Compil32.exe)
  • website.properties with the keys server (JPhotoTagger's web site server), user and pass (login data, not very secure, restrict access!)

To create the Windows setup file, Inno Setup is required. The PDF manual creation requires a Bash interpreter (and some Unix utils), html2ps and ps2pdf. The PDF manual task should be executed through a platform independend Java library or program (TODO: Find a freeware solution which can be distributed side by side with the Ant file).

If all files are created or modified, they will be uploaded to the correct places, currently the ZIP and EXE file to Google Code and the other files to JPhotoTagger.org.
